# Account 无活动监控方案说明

## 业务需求
1. **60天无活动**：发送提醒邮件给 Account Owner
2. **90天无活动**：发送升级邮件给 Cindy

## 实现方案

### 第一步：创建自定义字段

在 Account 模块创建 **2 个字段**：

1. **Last_Activity_Date**
   - 字段类型：Date
   - 用途：存储最后活动日期

2. **Days_Since_Last_Activity**
   - 字段类型：Number
   - 用途：存储无活动天数

**创建路径**：
```
设置 → 自定义 → 模块 → Accounts → 字段 → 新建字段
```

## 关键技术点

### 1. 正确的 API 使用

✅ **使用正确的 API**：
- `zoho.crm.getRelatedRecords("Tasks", "Accounts", accountId)`
- `zoho.crm.getRelatedRecords("Calls", "Accounts", accountId)`
- `zoho.crm.getRelatedRecords("Events", "Accounts", accountId)`

❌ **不要使用错误的 API**：
- ~~`zoho.crm.getRelatedRecords("Activities", ...)`~~ - 返回 204
- ~~`zoho.crm.getRelatedRecords("ClosedActivities", ...)`~~ - 返回 400

### 2. 日期排序

✅ **使用降序排序**（最新的在前面）：
```deluge
sortedDates = allActivityDates.sort(false);  // false = 降序
lastActivityDate = sortedDates.get(0);  // 第一个就是最新的
```

### 3. 自动分页

✅ **自动从上次停止的地方继续**：
```deluge
// 从 CRM Logs 读取上次处理到的页码
lastProcessedPage = 0;
lastLog = zoho.crm.searchRecords("Logs", "(Name:starts_with:Account_Activity_Sync_Page_)");
// 找到最大的页码
startPage = lastProcessedPage + 1;
```

**包含**：
- ✅ Tasks（任务）

## 常见问题

### Q: 如何知道处理到第几页了？
A: 查看 CRM Logs 模块，搜索 `Account_Activity_Sync_Page_`，找到最后一条记录的页码。或者直接查看控制台输出的"下次执行将从第 X 页开始"。

### Q: 如果某个 Account 更新失败怎么办？
A: 日志中会记录失败的 Account ID 和错误原因。可以单独处理这些失败的 Account。

### Q: 系统字段 `Last_Activity_Time` 为什么不准确？
A: 该字段包含 Notes、Emails 等非活动记录，不符合业务需求。我们使用 Tasks、Calls、Events 计算准确的最后活动时间。

### Q: 如何修改每次处理的页数？
A: 在 `Sync_All_Accounts_Auto.deluge` 中找到 `maxPages = 10;` 这一行，修改为你想要的页数。

### Q: 如何修改 Cindy 的邮箱？
A: 在 `Weekly_Check_Accurate.deluge` 中找到 `cindyEmail = "cindy@yourcompany.com";` 这一行，修改为实际邮箱地址。
