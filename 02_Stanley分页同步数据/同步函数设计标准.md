# 同步函数设计标准

## 一、函数命名规范

### 1. 同步单个记录
- **命名格式**：`SyncSingle{Module}`
- **示例**：`SyncSinglePO`, `SyncSingleInvoice`, `SyncSingleVendorBill`
- **类型**：`string standalone.SyncSingle{Module}(String recordId)`

### 2. 批量同步（带结果返回）
- **命名格式**：`Sync{Module}WithResult`
- **示例**：`SyncPOWithResult`, `SyncInvoiceWithResult`
- **类型**：`string standalone.Sync{Module}WithResult(Int startOffset)`
- **参数**：`startOffset` - 起始偏移量
- **返回值**：Map 包含 `totalProcessed`, `totalSuccess`, `totalFailed`, `finalProcessed`, `hasMore`, `errorIds`

### 3. 全量同步（调度器）
- **命名格式**：`SyncAll{Modules}`
- **示例**：`SyncAllPOs`, `SyncAllInvoices`
- **类型**：`void schedule.SyncAll{Modules}()`
- **职责**：
  - 从日志模块读取同步进度
  - 调用 `Sync{Module}WithResult(startOffset)`
  - 保存新的同步进度
  - 提示设置自动调度

### 4. 最新数据同步
- **命名格式**：`SyncNewest{Module}`
- **示例**：`SyncNewestPO`, `SyncNewestInvoice`
- **类型**：`void schedule.SyncNewest{Module}()`
- **职责**：
  - 从日志模块读取最后同步日期
  - 根据日期过滤数据
  - 批量同步

## 二、日志模块使用规范

### 1. 日志模块分类

| 模块类型 | 使用场景 | 示例模块 |
|---------|---------|---------|
| **vendorLogs** | Vendor 相关模块 | Purchase_Orders, VendorBill, VendorPayment, VendorCredit, ItemReceipt |
| **otherLogs** | 其他业务模块 | Invoice, CashSale, SalesOrder |
| **SOLogs** | SalesOrder 专用 | SalesOrder |
| **Logs** | 通用日志（旧版，逐步废弃） | Customer, Product |

### 2. 日志记录类型

#### 类型 A：同步状态记录
- **Name**: `{Module}SyncStatus`
- **Modules**: `{ModuleName}`
- **Log_Content**: JSON 格式
  ```json
  {
    "currentOffset": 220,
    "lastSyncTime": "23-Jun-2026 11:02:40",
    "hasMore": true
  }
  ```

#### 类型 B：错误记录
- **Name**: `{Module}Sync_400Errors_{startOffset}_{finalProcessed}`
- **Modules**: `{ModuleName}`
- **Error_Ids**: 错误ID列表（JSON字符串）
- **Log_Content**: 批次信息
- **Sync_Result**: 统计信息（可选）

**命名示例：**
- `POSync_400Errors_0_220`
- `InvoiceSync_400Errors_440_660`
- `VendorBillSync_400Errors_220_440`

## 三、数据同步流程

### 标准两阶段流程

#### 阶段 1：调度函数（schedule）
```javascript
void schedule.SyncAll{Modules}()
{
    // 1. 从日志模块读取进度
    currentOffset = 0;
    statusNotes = zoho.crm.searchRecords("{LogModule}","(Name:equals:{Module}SyncStatus)");
    noteModule = statusNote.get("Modules");
    if(noteModule == "{ModuleName}")
    {
        currentOffset = statusMap.get("currentOffset");
    }
    
    // 2. 调用同步函数
    syncResult = standalone.Sync{Module}WithResult(currentOffset);
    
    // 3. 保存进度
    statusData.put("currentOffset", finalProcessed);
    statusData.put("lastSyncTime", zoho.currenttime.toString());
    statusData.put("hasMore", hasMore);
    statusNoteParams.put("Name", "{Module}SyncStatus");
    statusNoteParams.put("Modules", "{ModuleName}");
    
    // 4. 提示自动调度
    if(hasMore == true)
    {
        info "请在Zoho CRM Workflow中设置自动调度，间隔5分钟";
    }
}
```

#### 阶段 2：同步函数（standalone）
```javascript
string standalone.Sync{Module}WithResult(Int startOffset)
{
    // 1. 配置参数
    limit = 1000;
    maxProcessCount = 220;  // 每次最多处理220条
    
    // 2. 分页获取数据
    // 3. 逐条同步
    // 4. 记录错误ID
    
    // 5. 保存错误记录
    if(errorIds.size() > 0)
    {
        errorParams.put("Name", "{Module}Sync_400Errors_" + startOffset + "_" + finalProcessed);
        errorParams.put("Modules", "{ModuleName}");
        errorParams.put("Error_Ids", errorIds.toString());
        errorParams.put("Log_Content", "批次: " + startOffset + " - " + finalProcessed + "...");
    }
    
    // 6. 返回结果
    return result;
}
```

## 四、重要参数配置

### 1. 批量处理数量
- **推荐值**：220 条/批次
- **原因**：避免 API 频率限制和超时

### 2. API 分页大小
- **推荐值**：1000 条/页
- **说明**：NetSuite API 的 limit 参数

### 3. 自动调度间隔
- **推荐值**：5 分钟
- **说明**：给前一批次足够的处理时间

## 五、字段映射规范

### 1. vendorLogs 模块字段
- **Name**: 日志名称
- **Modules**: 模块名称
- **Error_Ids**: 错误ID列表
- **Log_Content**: 日志内容
- **Sync_Result**: 同步结果统计

### 2. otherLogs 模块字段
- **Name**: 日志名称
- **Modules**: 模块名称
- **Error_Ids**: 错误ID列表
- **Log_Content**: 日志内容

## 六、错误处理规范

### 1. API 错误
- 记录失败的记录ID
- 继续处理其他记录
- 批次结束后保存错误ID列表

### 2. 解析错误
- 捕获 JSON 解析异常
- 如果已收集数据，继续处理
- 记录错误信息到日志

### 3. 认证错误（401）
- 可能是 API 频率限制
- 保存当前进度
- 等待下次调度继续

## 七、代码复用

### 推荐：创建通用的错误保存函数
```javascript
void automation.saveErrorIdsToLogs(
    String errorIdsStr,
    Int errorIdsCount,
    Int startOffset,
    Int finalProcessed,
    Int totalSuccess,
    Int totalFailed,
    String moduleName,

```

**参数说明：**
- `errorIdsStr`: 错误ID列表（JSON字符串）
- `errorIdsCount`: 错误数量
- `startOffset`: 批次起始位置
- `finalProcessed`: 批次结束位置
- `totalSuccess`: 成功数量
- `totalFailed`: 失败数量
- `moduleName`: 模块名称（如 "Purchase_Orders", "Invoice"）
- `logModule`: 日志模块名称（如 "vendorLogs", "otherLogs"）

## 九、注意事项

1. **日志模块一致性**：同一个模块的所有同步函数必须使用同一个日志模块
2. **Modules 字段**：保存和读取时都要检查 `Modules` 字段
3. **错误日志命名**：使用 `{Module}Sync_400Errors_{start}_{end}` 格式
4. **同步状态命名**：使用 `{Module}SyncStatus` 格式
5. **批量大小**：不要超过 220 条/批次，避免 API 限制
6. **自动调度**：设置 5 分钟间隔，避免重叠执行
