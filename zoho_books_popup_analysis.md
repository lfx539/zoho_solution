/*
 * Zoho Books 采购订单自定义弹窗需求分析
 *
 * 需求：
 * - 采购订单新增自定义字段 "shipping_method"
 * - 选项：Sea Freight, To Customer, Air Freight
 * - 当选择 "Sea Freight" 时，弹出确认框
 * - 提示："海运要4-6周，是否可以接受"
 * - 提供两个按钮："接受" / "不接受"
 *
 * ========== 分析 ==========
 *
 * 一、Zoho Books 的限制
 * -------------------------
 * Zoho Books 与 Zoho CRM 不同，不支持直接在字段变更时弹窗。
 * 原因：
 * 1. Zoho Books 的 Deluge 脚本主要用于后端逻辑（自动化、工作流）
 * 2. 没有前端 UI 交互 API（如弹窗、确认框）
 * 3. 不支持自定义 Widget 挂载到字段上
 *
 * 二、可行方案
 * -------------------------
 *
 * 方案 A: 页面加载时检查（有限制）
 * - 使用自定义函数 + URL 监控
 * - 只能在页面加载时检查，无法实时响应字段变更
 *
 * 方案 B: 保存时验证 + 报错回滚
 * - 当保存采购订单时检测 shipping_method 字段
 * - 如果是 Sea Freight，弹出警告并阻止保存
 * - 用户需要选择其他选项才能保存
 *
 * 方案 C: 使用浏览器扩展 / 油猴脚本
 * - 在客户端层面拦截页面加载
 * - 监听字段变化，触发自定义弹窗
 * - 这是最灵活的方案，但需要额外部署
 *
 * 方案 D: Zoho Books Widget（仅企业版）
 * - 创建自定义 Widget
 * - 挂载到采购订单页面
 * - 有限制，不是所有组织都可使用
 *
 * ========== 推荐方案 ==========
 *
 * 首选：方案 B（工作流 + 自定义函数）
 * - 实现相对简单
 * - 不需要额外部署
 * - 用户体验：保存时被阻止并看到警告
 *
 * 备选：方案 C（油猴脚本）
 * - 可以实现真正的弹窗交互
 * - 需要在用户浏览器安装脚本
 *
 * ========== 方案 B 详细实现 ==========
 *
 * 1. 在 Zoho Books 创建自定义函数
 *    - 触发器：采购订单创建/编辑时
 *    - 检查 shipping_method 字段值
 *    - 如果是 "Sea Freight"，返回错误信息阻止保存
 *
 * 2. 错误信息格式：
 *    "⚠️ 海运需要 4-6 周，请确认是否接受？
 *     请更改 Shipping Method 为其他选项。"
 *
 * ========== 方案 C 详细实现（油猴脚本）==========
 *
 * 如果需要真正的弹窗交互（带接受/不接受按钮），只能使用浏览器脚本：
 *
 * 1. 创建一个油猴脚本 (Tampermonkey)
 * 2. 监听 Zoho Books 采购订单页面的字段变化
 * 3. 当 shipping_method 字段值变为 "Sea Freight" 时
 * 4. 弹出自定义 Modal（使用 HTML/CSS/JS）
 * 5. 根据用户点击执行后续操作
 *
 * 脚本伪代码：
 * ```javascript
 * // 监听字段变化
 * const shippingField = document.querySelector('[fieldname="shipping_method"]');
 * shippingField.addEventListener('change', function() {
 *     if (this.value === 'Sea Freight') {
 *         showConfirmModal('海运要4-6周，是否可以接受？');
 *     }
 * });
 *
 * function showConfirmModal(message) {
 *     // 创建 Modal HTML
 *     // 绑定接受/不接受按钮事件
 * }
 * ```
 *
 * ========== 总结 ==========
 *
 * | 方案 | 弹窗交互 | 需要额外部署 | 实现难度 |
 * |------|---------|-------------|---------|
 * | B (工作流) | ❌ 仅阻止保存 | ❌ | ⭐⭐ |
 * | C (油猴脚本) | ✅ 完全自定义 | ✅ 用户浏览器 | ⭐⭐⭐ |
 *
 * 如果你接受方案 B（保存时阻止+提示），我可以帮你编写 Zoho Books 函数代码。
